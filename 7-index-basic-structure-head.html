Index

- La primera página siempre tiene que llamarse "index.html".
- El index.html será la página inicial que el servidor buscará, por defecto, al momento de abrir o ejecutar un proyecto.
- Cuando no tenemos una página index.html, nosotros debemos guiar al servidor, manualmente, para indicarle cuál sería nuestra página de inicio. De lo contrario, se visualizará toda la estructura de los directorios de archivos que tenemos en el proyecto.

Si nuestra aplicación web (WebApp) tiene más páginas, como por ejemplo, "contacto.html", "logging.html", etc; estas páginas las vamos a ligar a la única página principal, que vendría a ser "index.html".

  • < !DOCTYPE html >
    Esta etiqueta le indicará al Navegador que la estructura que analizará, a continuación, será de HTML:5, y así es como entenderá todas las etiquetas de HTML:5.
  • < html lang="en">
    "Contenedor principal" / "Contenedor padre", este contenedor llevará todas las etiquetas y todo el contenido que tendrá nuestro proyecto.
    Esta etiqueta llevará un atributo especial que será (lang=" "). El atributo lang es el encargado de decirle al Navegador qué lenguaje tendrá nuestro proyecto, ya sea Inglés, Español, Francés, o cualquiera que sea.
    Un ejemplo del uso del atributo lang, es para que el navegador sepa en qué idioma está la página que visitamos, y si está en un idioma distinto al nuestro, la pueda traducir a nuestro idioma.
    El valor de la etiqueta lang, es determinado por el tipo de idioma que tendrá la página web, por ejemplo, si es una web en español, será "es", lang="es", o si es una página en inglés, será "en", lang="en", o si es una página en francés, será "fr", lang="fr", y así, sucesivamente.
  • < head > < / head >
    La etiqueta "head" es contenedor hermano de la etiqueta "body", porque estas dos etiquetas están en el mismo nivel de la estructura del DOM: Document Object Model.
    Aquí va todo lo que es importante y necesario para el navegador, para que pueda cargar el proyecto tal y como lo diseñamos. Esta sección del código NO ES VISUAL para el usuario final, o sea, todo lo que son dependencias, librerías externas, fuentes, css, etc; todo eso tiene que ir ligado dentro de la etiqueta < head > < / head >. En esta sección mandamos a llamar a fuentes externas.
  • < body > < / body >
    Como ya se mencionó, la etiqueta "body" es contenedor hermano de la etiqueta "head", porque ambos están en el mismo nivel de la estructura del DOM: Document Object Model.
    El usuario final verá todo lo que va dentro de la etiqueta "< body > < / body >", todo lo que es texto, imágenes, videos, animaciones, etc; todo el aspecto interactivo que tendrá nuestro proyecto con el usuario, irá dentro del body.
  • Tabuladores: Son los espacios que debemos dejar al momento de escribir cada etiqueta para generar una simetría cuando esribimos código, para tener una lectura limpia y una comprensión rápida, y el IDE o Editor de Texto nos pueda ayudar a visualizar las estructuras de cada etiqueta, indicándonos su apertura y cierre.
    - Hay etiquetas que tienen apertura y cierre, ejemplo, < body > < / body >, como también hay etiquetas que se autocierran, por ejemplo, < br / >
    - Como buena práctica, las etiquetas que no tienen etiquetas de cierre, ejemplo, "< meta />", deben llevar un slash/diagonal "/" antes del cierre.

Una analogía para entender la estructura entre el head y el body, sería como si fuera un cuerpo humano, ya que en head almacenaría todas las conexiones nerviosas que necesita nuestro cuerpo, pero que no son vizuales, y en el body iría todo lo que usamos en el día a día, como nuestras articulaciones, que es claramente visible para el resto de personas.


Dentro de la etiqueta "head" irá todas las etiquetas queson importantes para el navegador para que el proyecto pueda cargar bien.

meta

La etiqueta "< meta />" da cierta informacion al navegador, para que sepa como tratar a nuestro proyecto.

Tipos de etiquetas "meta":

  • < meta charset="" />
    charset=" " llevará un valor "UTF-8", < meta charset="UTF-8" />, para que el navegador pueda entender una lista específica de caracteres especiales, para poder interpretarlos correctamente. También existe UTF-16 y UTF-32 que agrupan otro tipo de caracteres, pero estos dos últimos valores que almacenan otras listas, no son muy usados.
    Con "UTF-8" también puedes agregar emojis directamente al texto de la página, y se renderizarán como emojis.

  • < meta name="" /> - < meta name="description" content="Texto que irá en la descripción de la web, al momento de aparecer en los resultados de búsqueda."/>
    - El atributo name=" " llevará un valor "description".
    El manejo del valor "description" es parte del rol del SEO, < meta name="description" />, con esta etiqueta podremos añadir una breve descripción de la página, para que aparezca en el resultado de búsquedas en el Navegador.
    Contribuirá para que el navegador pueda ayudar a que cuando las personas busquen cierta información, el Navegador pueda relacioniar las palabras con la descripción que colocamos cuando nuestra web sea visible en el resultado de la búsqueda.
    - El atributo content=" " será el encargado de almacenar la descripción que queramos que tenga nuestra página web, al momento de aparecer en los resultados de búsqueda del navegador.

  • < meta name="keywords" content="Aquí, irán, las, palabras, clave" />
    Sirve para definir las palabras clave que ayudarán a los motores de búsqueda a mostrar nuestra página web al usuario, de una manera fácil y rápida.

  • < meta name="robots" content="index,follow" /> - Para indicarle a los robots de los buscadores en general.
    < meta name="googlebot" content="index,follow" /> - Para indicarle, específicamente, a los robots del buscador de Google.
    Robots de posicionamiento de los navegadores
    Nos ayuda a indicar a los motores de búsqueda para que puedan indexar nuestra página web para poderlas posicionar entre las primeras páginas de la lista de resultados.
    * content="index,follow" autoriza a ayudar a posicionar nuestra página en los resultados de búsquedas que tengan que ver con el contenido de nuestra web.
    El valor "follow" es importante para activar el seguimiento de nuestro "index". Para dejar de seguir nuestro index, o para desactivar el seguimiento, se reemplaza por "unfollow", quedando como content="index,unfollow".

  • < title > < title / >
    Sirve para redactar el título del proyecto, el nombre que pongamos a la página. Este texto aparecerá en la pestaña de la página, en el navegador. Es importante colocar el título.

  • < meta name="viewport" content="width=device-width, initial-scale=1.0" />
    name="viewport" hace mención al tamaño o width completo de la pantalla. Quiere decir, el tamaño que se genera en tiempo real, al modificar estirándo el ancho de la pantalla, eso es un viewport.
    Es muy importante para los proyectos que son Responsive Design. Actualmente todos los proyectos que se están realizando y los proyectos que tenemos que realizar, son y serán 100% Responsive Design.
    Es importante asegurarnos que esa etiqueta meta exista en todos nuestros proyectos, ya que nos ayuda a moldear el tamaño del texto y las imágenes dependiendo del dispositivo desde donde lo estamos abriendo, ya sea en un dispositivo mobile, o desde un navegador en el Escritorio/desktop.
    Si no agregamos esa etiqueta en nuestro proyecto, al momento de abrir nuestro proyecto en un dispositivo mobile, el tamaño del texto se verá chiquito, y habrá una desconfiguración en la posición del texto y las imágenes, y eso es una muy mala experiencia para el usuario.

  • Es MUY IMPORTANTE asegurarnos que el usuario no pase por ninguna mala experiencia al momento de probar nuestra aplicación o WebApp.

  • < link rel="stylesheet" href="style.css">
    rel="stylesheet" - Indica que el documento que se cargará, será una hoja de estilos.
    href="style.css" - Indica la referencia de dónde se encontrará ese documento. Como valor, se coloca la ruta en donde se encontrará el documento.

  • TENEMOS QUE SER MUY DESCRIPTIVOS al momento de elegir los nombres de los documentos de texto donde se costruirá cada parte de nuestros proyectos.
