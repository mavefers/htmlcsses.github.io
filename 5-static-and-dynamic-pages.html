Páginas Estáticas y Dinámicas

Diferencia entre Páginas Estáticas VS. Páginas Dinámicas

Páginas Estáticas:

Las páginas estáticas son todos los productos web que tienen una información predeterminada, que solo sea para consumir, leer, visualizar, y que esta información no va a cambiar, siempre será la misma.

    Ejemplos:
  • Un Blogpost
  • Una Biografía
  • Instrucciones
  • Manuales
  • Recetas
  • Informacion sobre un lugar o persona
Son artículos que una vez liberados, no se les harán un cambio alguno.
A estas páginas también se les conoce como Páginas Informativas o Landing Pages, que son puras páginas de información.
Ejemplo:
Página de empresa que tienen sus productos, que quiere comunicar mediante su página web: quienes son, qué búscan, qué hacen, su historia, y el equipo que los conlleva. La información brindada ráramente cambiará, para: agregar personas que han ingresado al equipo, para quitar los nombres de las personas que se han retirado del equipo, cambios en la administración, o modificación en la lista de productos ofrecidos, etc; pero gran parte de la información raramente va a cambiar, la mayor parte de lo que se publica será estando ahí, de forma estática, por mucho tiempo.
Probablemente, el cliente con el que estuviste trabajando te vaya a contactar nuevamente para solicitarte hacer algunos cambios, pequeños cambios, como cambiar la imagen, actualizar algunos datos de información o datos de contacto, etc; pero son cambios pequeños, la información será la misma.
Lo mejor, es que estas páginas no están conectadas a una servidor o alguna base de datos. No se necesita generar interacción con la base de datos porque no hay algo que genere algo del lado del usuario de la Web, no habrá algún tipo de logging o interacción similar entre la Web y el Usuario, por eso el tipo de información no cambiará y todo se generará desde el lado del servidor.

Los Post son páginas estáticas que no cambian, los Blogpost son webs que almacenan y generan posts o artículos, de manera independiente, quiere decir que cada post será de temas diferentes, no estarán muy relacionados, y no serán editados con el pasar del tiempo.


Páginas Dinámicas:

Tambien conocidas como WebApps. Son productos que ya dejan de tener el concepto de Web, para pasar al concepto como de Aplicaciones, por todas las cosas que se podrá realizar con estas mismas.

Estas Páginas Dinámicas o WebApps, sí estarán conectadas a una base de datos, porque nosotros, o el usuario a través del navegador, sí podremos generar una interacción con la página web o Aplicación, y por ende se tendrá que guaradar cierta información o progreso de lo que estamos realizando.

    Algunos ejemplos de interacción que podremos tener con las WebApps:
  • Guardar mi logging.
  • Crear una lista de productos.
  • Agregar productos a mi "Carrito".
  • Modificar mi perfil de usuario.
  • Dejar una reseña de algún producto.
  • Guardar en la Base de Datos la información de la lista de personas o usuarios que vayan dejando sus datos de contacto, para posteriormente poder contactarnos con ellos.

Como podremos observar, esta lista de ejemplos de interacciones con las WebApps nos demuestra la importancia del uso de la Base de Datos.

El enfoque para decidir el tipo de producto de página web, ya sea Web Dinámica o Web Estática, está en el tipo de usuario que tendrá la web, dependiendo de los que esté buscando el cliente.

Si el cliente solo quiere tener algún tipo de presencia en internet, subiendo datos que realmente no van a cambiar, lo ideal sería ofrecer una Páginas Estáticas.
Las páginas Estáticas también pueden tener NAVEGACIÓN entre diferentes tipos de segmentos o artículos que están dentro de la página en general. Esto hace que vayamos saltando de página en página, pero la información de cada una de las páginas seguirá siendo Estática, porque no va a cambiar, y no generá una interacción con el cliente.
Si es un cliente o empresa que quiere tener interacción, una base de datos de los usuarios y demás, en este caso, una página Dinámica sería la mejor opción.

    • - Maneja tecnologías específicas tanto Backend como Frontend, pero se especializa en un lado en particular.
  • ¿Qué tecnologías maneja principalmente un desarrollador Backend?
    • - Lenguajes de programación/frameworks que corran del lado del servidor, infraestructura en la nube y bases de datos.
  • ¿Qué tecnologías maneja principalmente un desarrollador Frontend?
    • - Estándares web: HTML, CSS y JavaScript.
  • HTML da la estructura a la información que ven los usuarios y CSS el estilo a esta información ¿Qué analogía los representa mejor?
    • - HTML es el esqueleto de una persona y CSS su apariencia al exterior.
