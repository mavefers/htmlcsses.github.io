Perfil del "Frontend"

(Cliente)

Es el desarrollador que maneja las cosas del lado del cliente.
A los Navegadores se les conoce como el "Cliente".

Frontend es la parte de un programa o dispositivo a la que un usuario puede acceder directamente.
Son todas las tecnologías de diseño y desarrollo web que corren en el navegador y que se encargan de la interactividad con los usuarios.
Un programador Frontend debe saber de código que entienda el navegador (HTML, CSS y JavaScript) para poder usar algunos frameworks o librerías que expanden sus capacidades para crear cualquier tipo de interfaces de usuarios. React, Redux, Angular, Bootstrap, Foundation, LESS, Sass, Stylus y PostCSS son algunos de ellos.


Tecnologías que trabaja un Front-end

Hay tres estándares que entiende el navegador:

  • HTML
  • CSS
  • JS


Frameworks de CSS:
Son ciertos fragmentos de CSS ya construidos, listos para que los agreguen a los proyectos y las personas puedan hacer las cosas de manera más rápida.
Los Frameworks otorgan cierto CSS construído para recortar los tiempos de creación de un producto.

  • Bootstrap
  • Tailwind CSS
  • Bulma
  • Foundation
  • pico.css


Frameworks y librerías de JS:
Después de que un front-end ya entiende bien el manejo de JavaScript, podrá elegir qué Framework o librería va a usar para poder desarrollar productos que puedan escalar más rápido y que puedan tener una interacción muchimo mayor.
Dependiendo del tipo de producto que se quiera construir, los front-ends eligen 1 o más librerías y frameworks.

  • React
  • Vue.js
  • Node.js
  • MongoDB
  • AngularJS
  • expressJS
  • ((Stack M.E.R.N.))


Procesadores de CSS:
Es una forma diferente de hacer CSS, tiene reglas distintas.
Algunos desarrolladores dicen que es como manejar CSS con super-poderes.
No son muy necesarios, ya que CSS ha estado evolucionando hasta igualar las funcionalidades de los Preprocesadores.
Pero es importante, ya que son muy demandados en las vacantes de trabajos en Front-end.

  • SASS (primer más demandado)
  • Stylus (segundo más demandado)
  • {less} (tercer más demandado)


Compiladores/Empaquetadores de JavaScript:
Como buena práctica, todos los desarrolladores Front-ends tienen que aprender las últimas actualizaciones para implementarlas en sus proyectos.

  • Babel
    Nos permite utilizar las últimas funcionalidades que tiene las últimas versiones de JavaScript, como ECMA10, ECMA11, ECMA12, compilándolas para utilizarlas en Navegadores no actualizados, para que estos Navegadores las puedan entender.
  • Webpack (empaquetador)
    Nos ayuda a generar empaquetados de ciertas tecnologías y dependencias en nuestros proyectos para trabajar un solo empaquetado, el cual será el que se mandará a producción, con todo el proyecto listo, para que corra directamente en el navegador.
