Existen dos tipos de etiquetas dentro de "body":

  • Etiquetas contenedoras:

    Las etiquetas contenedoras son etiquetas que van a llevar más etiquetas por dentro, y nos van a ayudar a generar una estructura de nuestra página web, tal y como lo pudimos ver en la anatomía de nuestr página web.


  • Etiquetas de contenido:

    Son las etiquetas que van a llevar consigo el texto, las imágenes, vídeos, animaciones, y cualquier cosa que se vaya a renderizar en nuestro proyecto.
    Algunas de estas etiquetas son:

    1. < header > < header / >
      - La parte del encabezado de nuestra página web.
      - Sección superior de nuestra página web.

    2. < nav > < /nav >
      - Es la barra de navegación, comúnmente está dentro del 'header'.
      - Por dentro pueden ir bototes de búsqueda, barras de input text, número de página, en general, todo lo que tenga relación con la navegación de la web.

    3. < main > < /main >
      - Es la parte principal de la página web, parte en la que va el contenido principal.
      - El "main" está fuera del 'header', por debajo del 'header'.
      - Le indica al navegador, que después del encabezado y de la barra de navegación, irá todo el contenido principal.
      • < section >< / section >
        - Por dentro, el main está fragmentado por "< section > < / section >"
        • < article> < / article >
          - Para fragmentar las secciones, tenemos la etiqueta article.
          - Podemos tener una sección con diversos artículos.

      • LISTAS:
        Para trabajar con un contenido que lleve listas, por ejemplo, una lista de artículos, una lista del super, o de cualquier cosa, tenemos dos maneras para hacerlas, < ol > < / ol > y < ul > < / ul >.
        Estas dos etiquetas nos van a dar alistas diferentes, dependiendo del tipo de listas que sean:
        < ol > < / ol >: ordered list (listas ordenadas)
        Con esta etiqueta se generarán listas ordenadas, todos los items puestos acá serán enumerados, desde el "1.", sin un límite que defina su finalidad, podemos ponder todos los items que queramos.
        < ul > < / ul >: unordered list (listas desordenadas)
        La lista que generemos con esta etiqueta, será una lista desordenada. Los items que estarán dentro de esta lista, se enlistarán con signos popularmente llamados "bullets", ya sean circulos, cuadrados y discos. Para definir cada tipo de bullet que se quiera usar, al momento de escribir el '< li >< / li >', como atricuto, se agrega type=" ", '< li type=" " > < / li >', y para definir el tipo de bullet, el valor cambiará dependiendo de cada tipo, por ejemplo.
        - Tipo círculo, type="circle"
        - Tipo cuadrado, type="square"
        - Tipo esfera vacía, type="disc"
        Dentro de ambos tipos de listas, los items que serán enlistados se harán con la etiqueta '< li > < / li >', que significa "list item".

      • Algunas etiquetas de contenido de texto:
        < p > < / p >
        Etiqueta para generar un párrafo
        < h1 > < / h1 >: encabezados
        Solo se usa una vez por página, esta regla es obligatoria.
        Se utiliza para hacer títulos.
        Hace el texto más grande que se pueda generar.
        < h2 > < / h2 > - < h6 > < / h6 >
        Se utiliza para hacer títulos.
        Existe la lista de etiquetas para definir títulos, el tamaño del texto va a depender del número 1 al 6, siendo 1, el tamaño más grande, y 6, el tamaño más pequeño.
        < a href=" "> < / a >
        Etiqueta "ancla".
        Nos ayuda a generar hipervínculos.
        Este es un "ancla". < se utiliza un '#' para que cuando aún no se haya definido qué referencia, href tendrá, al hacer click, no nos lleve a ningún lugar, o nos nos redirija a ninguna otra página.

    4. < footer > < / footer >
      - Es el apartado que contendrá todo lo que corresponde al "pié de página".

Todo lo que se ha redactado anteriormente viene siendo parte de la semántica de HTML, y nosotros, como buena práctica tenemos que utilizar HTML semántico.
Esto quiere decir que, para cada tipo de acción tiene una etiqueta específica, por ejemplo, para definir la parte del encabezado, sí o sí se tiene que crear con la etiqueta '< header > < / header >', para así, para cada tipo de acción que se quiera realizar.
